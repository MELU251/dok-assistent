<objective>
Add module-level singleton getter functions to src/ingest.py and src/retrieval.py so that
OllamaEmbeddings and the Supabase client are created once and reused for every subsequent call.
Update the affected functions to use these getters. Update test_ingest.py to patch at the
getter level (not the class level) to account for the lru_cache pitfall.

embed_and_store/search/delete call opens new HTTP connections to the Ollama VPS and Supabase,
exhausting the connection pool under sustained load.
Output: Two updated source files with singletons. One updated test file. All singleton tests green.

lru_cache pitfall (from RESEARCH.md Pitfall 2):
After adding singletons, test_ingest.py's test_stores_correct_number_of_chunks must
patch the GETTER (src.ingest._get_embedder) not the class (src.ingest.OllamaEmbeddings),
because the lru_cache may already hold a real instance from a prior test run.
The correct patch after this plan:
```python
@patch("src.ingest._get_supabase_client")
@patch("src.ingest._get_embedder")
@patch("src.ingest.get_settings")   # can be removed if settings not called directly
def test_stores_correct_number_of_chunks(
    self, mock_settings, mock_get_embedder, mock_get_client, sample_chunks
):
    mock_embedder = MagicMock()
    mock_embedder.embed_documents.return_value = [[0.1] * 768] * len(sample_chunks)
    mock_get_embedder.return_value = mock_embedder
    # ... rest unchanged
```
