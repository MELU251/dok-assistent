## Task Details

### Task 1+2: `src/extractor.py`

**AngebotData-Modell:**
```python
class AngebotData(BaseModel):
    title: str           # Titel / Thema des Lastenhefts
    summary: str         # Zusammenfassung des Anforderungsdokuments
    requirements: list[str]       # Explizite Anforderungen (was muss erfüllt werden)
    special_requests: list[str]   # Sonderwünsche / nicht-Standard-Anforderungen
```

**Extraction-Prompt:**
- System: Instruktionen zur strukturierten Extraktion + JSON-Schema
- User: Dokumenten-Kontext (Chunks als String)
- Claude antwortet mit einem JSON-Block → wird geparst
- Bei Parse-Fehler: ValueError mit klarer Meldung

**Cost-Tracking:** identisches Logging-Pattern wie `pipeline.py`

## Prompt Design

```
SYSTEM:
Du bist ein technischer Analyst. Analysiere das folgende Lastenheft und extrahiere
die Anforderungen strukturiert als JSON.

Antworte NUR mit einem JSON-Objekt in diesem Format (kein Markdown, kein Erklaerungstext):
{
  "title": "Kurztitel des Projekts/Produkts (max. 80 Zeichen)",
  "summary": "Zusammenfassung was benoetigt wird (2-4 Saetze)",
  "requirements": ["Anforderung 1", "Anforderung 2", ...],
  "special_requests": ["Sonderwunsch 1", ...]
}

Regeln:
- requirements: Pflichtanforderungen, die erfuellt werden MUESSEN
- special_requests: Wuensche, optionale Features, Nice-to-haves (kann leer sein)
- Antworte auf Deutsch
- Keine Felder auslasen; special_requests darf [] sein

USER:
Lastenheft:
{context}
```
