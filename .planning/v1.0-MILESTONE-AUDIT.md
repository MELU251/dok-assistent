## Cross-Phase Integration Findings

### Issue 1 — WORK-06: Dead Orchestration Function
- `src/workflow.py:create_angebotsentwurf()` is imported at `app.py` line 24 but never called
- `_run_workflow_flow()` + `_run_workflow_generation()` reimplement the pipeline inline
- User story (upload → extract → HIL → generate → download) is functionally complete via inline code
- Risk: bug fixes in `create_angebotsentwurf()` don't propagate to production path

### Issue 2 — TECH-02/TECH-03: Dual Singletons
- `src/ingest.py` and `src/retrieval.py` each define independent `_get_embedder()` + `_get_supabase_client()` caches
- At runtime: 2× OllamaEmbeddings instances, 2× Supabase clients in process
- Phase 01 intent was one process-wide singleton; this is two per-module singletons
- Functionally harmless for the PoC

### Issue 3 — WORK-04: HIL Hint Delivery
- `generate_angebot()` returns `hil_hinweis` key — embedded in `.docx` via `_write_hil_notice()`
- `_run_workflow_generation()` never reads `hil_hinweis` to display it in the Chainlit chat
- If requirement intent was also a chat-visible warning, this is a gap

### Issue 4 — UI-02: Document List During Session
- `get_indexed_documents()` is called only in `_build_welcome_content()` (session start)
- Requirement says: "jederzeit im laufenden Chat" (anytime during chat)
- No `/dokumente` or equivalent command exists for mid-session document listing
